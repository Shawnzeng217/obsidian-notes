## 🏗️ 微信小程序基础知识

### 小程序是什么？

| 特性 | 说明 |
|------|------|
| **平台** | 运行在微信内，无需下载安装 |
| **技术栈** | JavaScript + WXML + WXSS |
| **框架** | 原生框架 或 Vue/Taro/uni-app 等 |
| **生态** | 微信支付、用户授权、分享等内置能力 |
| **优势** | 快速启动、低成本分发、内置支付 |

### 核心组成

```
小程序项目结构：
├── pages/                    # 页面文件夹
│   ├── index/               # 页面目录
│   │   ├── index.js         # 页面逻辑
│   │   ├── index.wxml       # 页面结构（类似 HTML）
│   │   ├── index.wxss       # 页面样式（类似 CSS）
│   │   └── index.json       # 页面配置
│   └── detail/
├── components/              # 可复用组件
├── utils/                   # 工具函数
├── api/                     # API 请求封装
├── app.js                   # 应用主文件
├── app.wxml
├── app.wxss
├── app.json                 # 应用配置（页面注册、导航栏等）
└── project.config.json      # 开发工具配置
```

### 核心概念

#### 1. 页面生命周期
```javascript
Page({
  onLoad(options) {},       // 页面加载
  onShow() {},              // 页面显示
  onReady() {},             // 页面初次渲染完成
  onHide() {},              // 页面隐藏
  onUnload() {},            // 页面卸载

  // 自定义方法
  handleTap() {},
  // 数据
  data: {
    message: 'Hello'
  }
})
```

#### 2. 数据绑定
```wxml
<!-- 单向绑定 -->
<text>{{message}}</text>

<!-- 属性绑定 -->
<image src="{{imageUrl}}" />

<!-- 列表渲染 -->
<view wx:for="{{list}}" wx:key="id">
  {{item.name}}
</view>

<!-- 条件渲染 -->
<view wx:if="{{isShow}}">显示</view>
```

#### 3. 事件处理
```wxml
<button bindtap="handleClick" data-id="123">
  点击按钮
</button>
```

```javascript
Page({
  handleClick(e) {
    const id = e.currentTarget.dataset.id;
  }
})
```

#### 4. API 调用
```javascript
wx.request({
  url: 'https://api.example.com/data',
  method: 'GET',
  success: (res) => {
    console.log(res.data);
  },
  fail: (err) => {
    console.error(err);
  }
})
```

## 🤖 AI 在小程序开发中的应用

### 应用场景

#### 1. 代码生成
**AI 帮助生成组件、页面、API 封装**

```
需求描述 → AI → 完整代码
例：
"生成一个用户信息展示页面，包含头像、昵称、签名，有编辑按钮"
→ AI 生成 WXML + JS + WXSS
```

#### 2. 架构设计
**AI 帮助规划项目结构、API 设计、状态管理**

```
问题 → AI → 设计方案
例：
"我要开发一个购物小程序，如何组织代码？"
→ AI 提供文件夹结构、分层架构建议
```

#### 3. 集成 AI API
**在小程序中调用 Claude / GPT / Gemini 等服务**

```javascript
// 在小程序中调用 Claude API
async function askAI(prompt) {
  const response = await wx.request({
    url: 'https://api.anthropic.com/v1/messages',
    method: 'POST',
    header: {
      'Authorization': 'Bearer ' + CLAUDE_API_KEY
    },
    data: {
      model: 'claude-3-sonnet-20240229',
      messages: [{ role: 'user', content: prompt }]
    }
  });
  return response.data;
}
```

#### 4. 代码审查与优化
**AI 审查代码质量、性能问题、最佳实践**

```
代码 → AI 审查 → 改进建议
例：
"审查这个页面的性能瓶颈"
→ AI 识别问题并提供优化方案
```

#### 5. 文档生成
**自动生成 API 文档、使用指南、更新日志**

## 🎯 AI 辅助开发最佳实践

### Prompt 工程
```
好的 prompt 包含：
1. 角色设定：[你是一个微信小程序专家]
2. 任务描述：[我要创建...]
3. 具体需求：[功能1、功能2、功能3]
4. 格式要求：[代码要完整可用]
5. 约束条件：[微信小程序原生框架]

例子：
"你是一个微信小程序高级开发工程师。
我要创建一个天气查询小程序，需要实现：
- 首页显示当前城市天气
- 支持切换城市
- 显示7天预报
- 完整的 WXML、JS、WXSS

请生成可直接使用的代码。"
```

### 迭代策略
```
1️⃣ 生成基础版本
   "为我生成一个天气小程序首页"

2️⃣ 反馈与调整
   "现在加上城市切换功能"

3️⃣ 优化与增强
   "优化 UI，添加动画效果"

4️⃣ 完善细节
   "添加错误处理和加载状态"
```

### 常见问题处理
```
当遇到问题时：
1. 提供完整的错误信息
2. 说明你做了什么
3. 期望的结果是什么
4. 已经尝试过的方案

例：
"我的分享功能报错：wx.shareAppMessage is not a function
我在按钮的 tap 事件中调用了这个方法。
期望点击分享按钮时弹出分享菜单。
已尝试检查了文档，但还是不行。"
```

## 🔍 常见陷阱与解决方案

| 陷阱 | 原因 | 解决方案 |
|------|------|---------|
| **API Key 泄露** | 前端暴露密钥 | 使用后端代理，前端只调用自己的 API |
| **请求超时** | 网络问题或 API 响应慢 | 添加超时处理和重试机制 |
| **页面性能差** | 渲染大列表 | 使用虚拟列表（wx:for 优化）或分页 |
| **用户体验差** | 加载无反馈 | 添加 loading 动画、进度提示 |
| **兼容性问题** | 不同微信版本支持不同 | 检查 API 文档，添加容错 |
