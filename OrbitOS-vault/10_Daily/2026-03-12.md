### 朋友圈草稿：OrbitOS + Happy CLI 工作流介绍

---

**用 AI 帮我管理知识库，还能远程操控——这套工作流我用了一周，真的回不去了。**

分享一个我最近搭起来的个人知识管理工作流，核心是三个工具的组合：**Obsidian + Happy CLI + obsidian-git**。

---

**先说是什么**

**Obsidian** 大家可能不陌生——本地 Markdown 笔记软件，文件全在自己手上，支持双向链接、插件生态很成熟。我在里面搭了一套叫 **OrbitOS** 的结构：Inbox 捕获想法、Daily 日记、Project 管理项目、Wiki 存原子知识——所有内容都互相链接，像一个围绕自己运转的知识轨道。

**Happy CLI** 是一个让你用手机或任意设备远程连接本地 AI（Claude / Gemini）的工具。本地跑着 Claude Code，Happy 提供一个入口，走到哪里都能调用本地的 AI Agent，延迟低、不限上下文、还能直接操作本地文件。

**obsidian-git** 是 Obsidian 的 Git 插件，一键把 vault 同步到 GitHub，自动提交、自动 push，版本历史全留着。

---

**组合起来能做什么**

我给本地 Claude 写了一批自定义技能（slash commands）：
- `/start-my-day` — 早上打开手机，AI 自动回顾昨天、列出今日重点、拉取 AI 简报
- `/kickoff` — 有个想法，AI 帮你拆成结构化项目，直接写进 vault
- `/sync` — 一个命令，所有改动 commit + push 到 GitHub，就像你在本地操作一样

**举个真实的 demo：**

今天我在外面，用 Happy CLI 连上本地 Claude，让它在 Inbox 里创建一个测试笔记——它直接在我的 vault 里建了一个 `test-sync.md`，内容带好 frontmatter，然后我一句 `/sync`，几秒钟 GitHub 上就有了这条提交记录。全程没碰电脑。

---

**从 0 到 1 怎么配置**

1. **装 Obsidian**，建好你的 vault 结构（或者直接用 OrbitOS 模板）
2. **装 obsidian-git 插件**，在设置里绑定你的 GitHub 仓库（SSH key 或 HTTPS 均可）
3. **装 Claude Code / Gemini CLI**，在 vault 根目录放 `CLAUDE.md` 写好 AI 的行为规则
4. **安装 Happy**，登录后本地 Claude Code 就能被远程访问
5. 写几个自定义 skill（`.agents/skills/` 目录下），比如 `/sync`，让 AI 帮你执行 git 操作

配置完之后的日常：手机开 Happy → 跟 Claude 说"帮我记一个想法" → AI 建好笔记 → `/sync` 推到 GitHub → Obsidian 桌面端自动拉取更新。

---

**为什么我觉得这套东西值得分享**

不是因为它有多酷，而是因为它真的降低了"记下来"的摩擦。以前很多想法因为懒得打开电脑就消失了。现在有个 AI 帮你把碎片整理成结构，git 帮你做版本管理，手机就能操作——知识积累这件事终于变成了习惯而不是负担。

感兴趣的可以聊，工具链和 OrbitOS 结构我都可以分享。

### 想法：上下文感知的迷你宏键盘

**核心需求**
购买一个外接小键盘（1-3 个物理按键），每个按键可自定义为快捷组合键，并根据当前活动窗口自动切换映射规则，做到"一键多义"。

**可行性：完全可行** — 这是成熟的产品方向，硬件 + 软件组合即可实现。

---

#### 方案一：购买现成 Macro Pad（推荐）

**硬件选择**

| 产品 | 按键数 | 特点 | 参考价 |
|------|--------|------|--------|
| DOIO KB03-01 | 3键 + 旋钮 | 支持 QMK/VIA，淘宝有货 | ¥150–200 |
| 各类 AliExpress 宏键盘 | 1–4键 | 最便宜，基础够用 | ¥30–100 |
| Elgato Stream Deck Mini | 6键（带显示屏） | 软件最成熟，有屏幕提示 | ~$100 |
| Keychron Q0 | 5键 | 做工好，支持 QMK | ¥300+ |

**推荐购买：DOIO KB03-01 或类似支持 QMK/VIA 的 3 键 Pad**

---

#### 上下文感知（软件层）

硬件本身不需要感知窗口 —— 由 Mac 上的软件层来实现。

**macOS 方案：**

| 工具 | 费用 | 能力 |
|------|------|------|
| **Karabiner-Elements** | 免费 | 按 frontmost app 切换映射，功能强，配置灵活 |
| **Keyboard Maestro** | $36 | 最灵活，可按 app / URL / 时间触发宏 |
| **BetterTouchTool** | $22 | 支持 app 级别的按键动作 |
| Stream Deck 原生软件 | 免费（配套） | 内置 per-app Profile 切换 |

**推荐：Karabiner-Elements（免费）+ QMK/VIA 宏键盘**

---

#### 工作原理示意

```
按下物理按键 A
    ↓
Macro Pad 发出一个普通键（如 F13 或自定义 HID code）
    ↓
Karabiner-Elements 检测当前活动窗口
    ↓
VS Code → 发送 Alt+Enter（快速修复）
Obsidian → 发送 Cmd+K（插入链接）
Chrome → 发送 Cmd+L（地址栏聚焦）
其他 → 默认行为
```

---

#### 方案二：纯软件（零成本验证）

先用 Karabiner-Elements 把现有键盘的某个闲置键（如右侧 Cmd、F13–F19）配置成上下文感知宏，验证工作流是否顺手 —— 再决定是否买硬件。

---

#### 方案三：DIY（极客路线）

用 **Raspberry Pi Pico** + CircuitPython，焊 1–3 个按键，写 HID 脚本，完全定制。成本 ¥50 以内，但需要动手能力。

---

#### 结论与建议行动

1. **先验证**：用 Karabiner-Elements 在现有键盘上测试上下文映射逻辑
2. **再购买**：如果体验满意，买 DOIO KB03 或类似 QMK 宏键盘
3. **配置**：用 VIA 网页配置器设置键值，Karabiner 做 app 级映射

**关键词**：`macro pad`, `QMK`, `VIA`, `Karabiner-Elements`, `HID`
